# MindFlow AI - Intelligent Note Management System

MindFlow AI is a modern, full-stack note management application that leverages Generative AI and Vector Databases to organize, summarize, and conceptually search your thoughts. 

This project was built to demonstrate clean architecture, robust API design, and practical integration of Large Language Models (LLMs) in a real-world SaaS-like environment.

---

## 🏗️ Architecture Explanation

The application follows a highly decoupled **Client-Server Architecture**:

1. **Frontend (Presentation Layer)**: 
   - Built with **React**, **TypeScript**, and **Vite**.
   - Styled with **Tailwind CSS**, featuring a custom glassmorphism aesthetic with seamless Light/Dark mode toggling.
   - Manages state locally and communicates with the backend via secure, token-based Axios requests.

2. **Backend (Application & Integration Layer)**: 
   - A RESTful API built on **Laravel**.
   - Handles robust authentication via **Laravel Sanctum**.
   - Acts as a secure proxy and orchestrator for external AI services. It intercepts user requests, processes data, fetches AI embeddings/summaries, and coordinates between the relational and vector databases.

3. **Data Layer**:
   - **Primary Database (MySQL via Eloquent)**: Acts as the absolute source of truth for Users and Note text/metadata. 
   - **Local Vector Database**: Stores 768-dimensional text embeddings generated by Google's Gemini model in a local JSON flat-file. This allows for lightning-fast Semantic Search based on the *context* and *meaning* of the notes rather than strict keyword matching.

## 🛠️ Setup Instructions

### Prerequisites
- PHP 8.2+ and Composer
- Node.js 18+ and npm/yarn
- A Google Gemini API Key


### 1. Backend Setup (Laravel)
```bash
cd backend
composer install
cp .env.example .env
php artisan key:generate
```
Open the `.env` file and configure your database and AI keys:
```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=nvecta
DB_USERNAME=root
DB_PASSWORD=
GEMINI_API_KEY=your_gemini_api_key_here
```
Run the migrations and start the server:
```bash
php artisan migrate
php artisan serve
```
*The backend will run on `http://127.0.0.1:8000`.*

### 2. Frontend Setup (React/Vite)
```bash
cd frontend
npm install
```
Ensure your `frontend/src/services/api.ts` is pointing to `http://127.0.0.1:8000/api`.
Start the frontend development server:
```bash
npm run dev
```
*The frontend will run on `http://localhost:5174` (or 5173).*

## 🗄️ Database Schema

### `users` table
- `id` (Primary Key, BigInt)
- `name` (String)
- `email` (String, Unique)
- `password` (String, Hashed)
- `timestamps`

### `notes` table
- `id` (Primary Key, BigInt)
- `user_id` (Foreign Key -> users.id)
- `title` (String)
- `content` (Text)
- `summary` (Text, Nullable) - *Stores AI generated summaries*
- `vector_id` (String, Nullable) - *Maps the MySQL row to the exact Vector Point ID in the local database*
- `timestamps`

## 🤖 AI Tools & Prompts Used

### AI Tools Integrated
- **Google Gemini API**: Used as the core LLM engine.
  - `gemini-flash-latest`: Used for high-speed text summarization.
  - `gemini-embedding-2`: Used to convert note text into 768-dimensional float arrays (embeddings).
- **Local Vector Database**: Custom JSON-based vector storage system built to index the embeddings and perform Cosine Similarity distance calculations natively in PHP.

### Prompts Used

**1. Summarization Prompt:**
When a user clicks "Generate AI Summary", the backend constructs the following prompt and sends it to the Gemini Flash model:
> *"Provide a detailed and comprehensive summary of the following note content. Include a brief overview paragraph followed by bullet points for the key takeaways to capture all important details. Be thorough and insightful. Do not add introductory remarks: \n\n [USER_NOTE_CONTENT]"*

**2. Semantic Search Embedding:**
No strict "prompt" is used here, but the data is structured to maximize contextual accuracy before being fed into the embedding model:
> *"Title: [NOTE_TITLE] \n Content: [NOTE_CONTENT]"*
This vector is then stored locally. When the user searches, their query is also embedded and the system compares the vectors using Cosine Similarity.

## 📖 API Documentation

All endpoints are prefixed with `/api`. Postman collection format.

### Authentication (Public)
- `POST /register`: Accepts `name`, `email`, `password`, `password_confirmation`. Returns User & Token.
- `POST /login`: Accepts `email`, `password`. Returns User & Token.

### Notes CRUD (Protected - Requires Bearer Token)
- `GET /notes?page=1&search=xyz`: Fetches paginated list of notes. Supports wildcard standard searching.
- `POST /notes`: Creates a note (`title`, `content`). Generates vector embedding in background.
- `GET /notes/{id}`: Fetches specific note.
- `PUT /notes/{id}`: Updates note (`title`, `content`). Re-calculates and updates vector embedding.
- `DELETE /notes/{id}`: Deletes note from SQL and Vector DB.

### AI Endpoints (Protected)
- `POST /notes/{id}/summary`: Triggers Gemini to read the note content and save a generated summary. Returns the summary string.
- `GET /search?q=your_query`: Converts the query into a vector, queries the local database for nearest neighbors, and returns the strictly matching notes ordered by semantic similarity score.
